# Contributing to 3D Plane Game

Thank you for your interest in contributing to the 3D Plane Game! We appreciate your efforts in making this project better. Please follow the guidelines below.

## Code of Conduct
We expect all contributors to adhere to a code of conduct that promotes respectful interaction. Please be kind, respectful, and constructive in your feedback and interactions.

## Development Setup
1. **Clone the repository**:
   ```bash
   git clone https://github.com/manyamchintu1-netizen/3d-plane-game.git
   cd 3d-plane-game
   ```
2. **Install dependencies**:
   ```bash
   npm install
   ```
3. **Run the application**:
   ```bash
   npm start
   ```

## Coding Standards
- **JavaScript**: Follow [Airbnb JavaScript Style Guide](https://github.com/airbnb/javascript)
- **CSS**: Use [BEM (Block Element Modifier)](http://getbem.com/)
- **HTML**: Ensure semantic HTML is used and adhere to accessibility standards.

## Pull Request Process
1. Fork the repository.
2. Create a new feature branch:
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. Make your changes.
4. Commit your changes:
   ```bash
   git commit -m "Add some feature"
   ```
5. Push to the branch:
   ```bash
   git push origin feature/your-feature-name
   ```
6. Create a Pull Request.

## Testing Guidelines
- Write tests for your code. We use [Jest](https://jestjs.io/) for testing.
- Run tests before submitting the pull request:
   ```bash
   npm test
   ```

## Commit Message Conventions
- Use the imperative mood for commit messages. For example: "Add feature" instead of "Added feature".
- Include the type of change (feat, fix, docs, style, refactor) in the commit message.
  
**Example**: `feat: add new scoring system`
